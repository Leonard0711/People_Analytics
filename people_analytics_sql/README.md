# 📊 People Analytics con SQL (MySQL)

## 📌 Descripción
Este proyecto consiste en la **simulación de un análisis de datos en Recursos Humanos** utilizando **SQL (MySQL)**, a partir de un modelo de datos relacional construido con **tablas sintéticas**.

El objetivo es demostrar cómo, mediante consultas SQL, se pueden generar **potenciales insights** aplicables a la gestión del talento en un entorno empresarial.

---

## 🎯 Objetivos del proyecto
- Analizar el desempeño de empleados por área  
- Evaluar la rotación de personal  
- Identificar patrones de asistencia (tardanzas y ausencias)  
- Medir la antigüedad del talento  
- Explorar la distribución demográfica (género, edad)  

---

## 🛠️ Tecnologías utilizadas
- **MySQL**
- SQL:
  - Joins (INNER JOIN, LEFT JOIN)
  - Subqueries
  - CTEs (Common Table Expressions)
  - Window Functions
  - Funciones de agregación (AVG, COUNT, SUM)

---

## 🧩 Modelo de datos
El proyecto se basa en las siguientes tablas:

- **empleados**: información demográfica, salarial y organizacional  
- **desempeño**: evaluaciones de rendimiento  
- **asistencia**: registro de horas trabajadas, tardanzas y ausencias  
- **rotación**: información de bajas de empleados  

---

## 📊 Análisis realizados
Algunas consultas desarrolladas permiten:

- Calcular el desempeño promedio por área  
- Estimar la tasa de rotación anual  
- Identificar empleados con mayor número de tardanzas  
- Analizar la antigüedad promedio del talento activo  
- Comparar salarios dentro de cada área  
- Explorar relaciones jerárquicas dentro de la organización  

---

## 💡 Insights (potenciales)
Dado que los datos son sintéticos, los resultados representan **escenarios simulados**, pero permiten identificar:

- 🔁 Áreas con alta rotación → posibles problemas de clima laboral o crecimiento  
- ⏰ Tardanzas recurrentes → señales de compromiso o gestión del tiempo  
- 📈 Antigüedad del talento → niveles de estabilidad organizacional  
- ⚖️ Variaciones en desempeño → posibles diferencias en liderazgo o carga de trabajo  

---

## ⚠️ Consideraciones
> Este proyecto utiliza **tablas sintéticas sin datos reales**.  
> El enfoque está en el **diseño del modelo de datos** y la **lógica analítica de las consultas SQL**.
